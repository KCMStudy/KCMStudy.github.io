  • 전체 진행률 보는 법.
TABLE
sum(rows.done) AS "완료된 수",
length(rows) AS "총 수량",
round(sum(rows.done) / length(rows) * 100, 2) + "%" AS "진행율"
FROM
"/"
WHERE typeof(done) = "number"
GROUP BY "/" AS "전체"
  • 해설
    • [보고싶은 항목들] : GROUP BY 를 사용해서 rows라는 그룹핑된 데이터를 사용함.
    • Sum(rows.done) AS "완료된 수" : sum() 리스트의 합계 구하는 함수임.(rows.done 같은 것끼리 묶은 것의 done 값들이 리스트의 형태로 출력.)
    • length(rows) AS "총 수량" : length(rows) 하면 같은 것 끼리 그룹핑된 데이터의 길이를 보겠다는 뜻임. rows는 그룹핑된 데이터를 사용하는 거라서..
    • round(sum(rows.done) / length(rows) * 100, 2) + "%" AS "진행율" : 완료된 수 / 총 수량 * 100으로 진행률을 구한다.
      • round(값, 2) : 소수점 2자리까지 나오게 반올림.
      • "%" : 퍼센트 단위로 숫자뒤에 %가 나오게..
    • FROM "/" : 전체 폴더에서 보기
    • WHERE typeof(done) = "number" : done 이라는 값이 존재하는 조건만 보기 [1]
    • GROUP BY "/" AS "전체" : 같은 것들을 묶어서 보기. ("/"로 처리하면 전체 진행률이 나온다.) [2]
  • 도서별 진행율
TABLE
sum(rows.done) AS "완료된 수",
length(rows) AS "총 수량",
round(sum(rows.done) / length(rows) * 100, 2) + "%" AS "진행율"
FROM
"/"
WHERE typeof(done) = "number"
GROUP BY split(file.folder, "/")[0] AS "도서"
  • 해설
    • GROUP BY split(file.folder, "/")[0] AS "도서"
      • 같은 것만 묶어서 보겠다.
    • file.folder
      • 해당 파일의 폴더가 나옴, 파일 명 제외
    • split(file.folder, "/")
      • 파일의 폴더구조가 도서명폴더/챕터명폴더/도서정리.md일 경우 "도서명폴더", "챕터명폴더"로 폴더 명을 출력
    • split(file.folder, "/")[0]
      • split(file.folder, "/")에서 나온 결과에서 첫번째만 사용하는 역할. (도서명)
  • 챕터별 진행률
TABLE
sum(rows.done) AS "완료된 수",
length(rows) AS "총 수량",
round(sum(rows.done) / length(rows) * 100, 2) + "%" AS "진행율"
FROM
"/"
WHERE typeof(done) = "number"
GROUP BY reverse(split(file.folder, "/"))[0] AS "챕터"

  • GROUP BY reverse(split(file.folder, "/"))[0] AS "챕터"

    • file.folder
      • 해당 파일의 폴더가 나옵니다. 파일이름제외
    • split(file.folder, "/")
      • "/" 를 기준으로 쪼개겠다는 의미 입니다.
      • 나온 결과 값으로 리스트가 나옵니다
      • ["도서명폴더", "챕터명폴더"]
    • reverse(split(file.folder, "/"))
      • 리스트의 순서를 뒤집겠다. 리스트의 맨앞이 챕터명이 됩니다
      • ["챕터명폴더", "도서명폴더"]

  • reverse(split(file.folder, "/"))[0]

  • reverse(split(file.folder, "/"))에서 나온 결과에서 첫번째만 사용합니다.

  • 여기서는 챕터명이 됩니다.

  1. 모든 문서에 done 이 있는 것이 아니라 일부만 있기 때문에 done 타입을 확인하는 typeof(done) 이렇게 했고요 0 혹은 1만 적어놔서 타입이 number 인것만 보겠다고 조건을 넣었습니다.↩︎

  2. GROUP BY 를 사용하게 되면

    1. 테이블에 첫번째 항목으로 됩니다.
    2. [보고싶은 항목들] 에서 rows 라는 그룹핑된 데이터를 사용할 수 있습니다.
